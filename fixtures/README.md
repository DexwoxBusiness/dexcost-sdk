# Cross-SDK Fixture Suite

Single source of truth for cross-language parity. Every SDK (Python, Go, TypeScript,
Rust) must produce identical results against these fixtures.

## Why this exists

The cross-language audit (May 2026) found that the four SDKs silently diverge:
- Rust dispatched compute pricing on `ec2_share` while others used `ec2` (the entire
  IaaS share line went to $0 on cross-SDK pricing).
- Go's bundled JSON schema rejected GPU event types.
- TypeScript accumulated cost in IEEE-754 floats, drifting from Python's Decimal.
- Timestamp wire format differed in all four SDKs.

These bugs existed because each SDK tested itself against itself. This corpus is
what cross-SDK testing was missing.

## How each SDK uses this

Each SDK has a parity test (`test_cross_sdk_parity.py` / `cross_sdk_parity_test.go`
/ `cross-sdk-parity.test.ts` / `tests/cross_sdk_parity.rs`) that:

1. **Deserialization round-trip** — reads every `events/*.json` and `tasks/*.json`,
   parses into the SDK's typed model, re-serializes, asserts the output matches
   `expected_outputs/canonical_serialization/`.
2. **Pricing parity** — reads every `pricing_inputs/**/*.json`, runs through the SDK's
   pricing engine, asserts the resulting `cost_usd` matches
   `expected_outputs/pricing/` (Decimal-aware comparison; not float).
3. **Security redaction** — reads every `events/edge_cases/url_*.json`, runs the URL
   through the SDK's scrubber, asserts the output matches
   `expected_outputs/security/`.

All three assertions MUST pass before any SDK PR can merge. CI failure on any of
these blocks the cross-SDK release.

## How to regenerate `expected_outputs/`

Only Python is canonical (it has been the source of truth since Phase 1). To update:

```bash
cd dexcost-sdk
DEXCOST_REGENERATE_FIXTURES=1 python -m pytest python/tests/test_cross_sdk_parity.py -k regenerate
```

The script writes new files into `expected_outputs/`. Inspect the diff before
committing. Any change to `expected_outputs/` requires updating ALL FOUR SDKs in the
same PR — never check in an output drift without aligning the downstream SDKs.

## Deterministic IDs

All UUIDs in the corpus are fixed (not random). Convention:
- workspace_id: `00000000-0000-0000-0000-000000000001`
- task_id: `11111111-XXXX-XXXX-XXXX-XXXXXXXXXXXX` (each task gets a distinct second segment)
- event_id: `22222222-XXXX-XXXX-XXXX-XXXXXXXXXXXX`

Timestamps are also fixed (no `Date.now()`); see `2026-05-26T12:34:56.789012+00:00`
as the canonical instant for events unless a fixture deliberately tests time drift.

## Versioning

Files are suffixed `.vN.json`. When the wire format breaks, add v2 fixtures
alongside v1. Existing fixtures stay in place so old SDKs can still validate
against them.
